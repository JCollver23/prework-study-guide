HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • HTML is the framework of the website.
  • HTML elements are compsed of an opening tag and a closing tag, represented with angle brackets.
  • Common elements are: text elements, image elements, header elements and list elements

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • Cascading style sheet CSS - defines how website content is preseted to the user
  • 3 ways to style a webpage in CSS: inline CSS, internal CSS style sheet, and external CSS style sheet
  • Inline allows you to add style to an element directly in HTML file using a style attribute with a CSS rule
  • Inline is useful for making quick and permanent changes
  • Internal CSS style sheet embeds an entire style sheet directly in HTML file using a STYLE element in the HEAD element
  • External CSS style sheet is preferred as it is keeps all rules in a separate file and is easy to change
  • External CSS style sheet is created in a external file and is linked using the LINK element in HTML's
  • Declarations contain the CSS property we want to apply, and the value of the property. i.e. property: color, value: blue

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git clone: retrieves a repository via URL
  • git branch: lists your branches
  • git merge: merge specified branch into the current or main branch
  • Complete the git flow - PR: check git status and confirm in main branch. Input git checkout main to switch to main branch if not already there. Git checkout -b "branch name" will create the new branch. Git add -A will add all of your commits. Git add -m "add your commit message". Git pull origin main initiates pull request to connect back to main branch. Git push origin "new branch name" will link the two for submitting pull request in github.

JavaScript

    • JavaScript is case sensitive. It is a high level dynamic programming language that allows users to interact with websites.
    • API - Application Programming Interfaces
    • A variable is a named container that allows us to store data in our code.
    • Variables are not directly assocaited with any particular value type as it can be assigned and reassigned.
    • Variable keyword "const" allows you to "lock down" a variable so that once it's value is declared it can never be reassigned.
    • Each line of JavaScript code ends with a semicolon ;
    • Console is a test environment. Log() is instructions that can be executed by a computer.
    • The console.log method makes it easy for developers to test and debug code.
    • String is a sentence of text. To signify that the value is a string, enclose with quoatation marks.
    • Numbers don't have quotes around them
    • Boolean is a true or false value, and does not need quaotations.
    • Array is a structure that allows you to store multiple values in a single reference.
    • Comments can be added between forward salsh asterisk and the closing asterisk forward slash. Comments are not visible.
    • Double forward slash also allows for adding comments in code where there is no line break.
    • Operators are mathmatical symbols that produce a result based on two values. EX: Addition +, Subtraction -, multiplication *, division /, assignment, strict equality, and Does-not-equal.
    • Conditionals are code structures used to test if an expresssion returns true or not. Commonly seen in "if...else" statments.
    • Control flow is the order in which a computer executes code in a script.
    • JavaScript code runs from the first statement at the top of the page, down to the last statement at the bottom of the page. It is linear and only deviates if conditions are met by using "if..." statement.
